##Git 사용법
###하루의 시작
1. git pull
2. git log
	- 작업 내역 확인 가능 (끝내는건 q)	
3. <- 너의 작업 ㄱㄱ ->
4. git add *
	- 변경 파일들을 모두(*) stage에 올리는 작업
5. git commit -m '메세지'
	- 실제 git에 올리기 직전(커밋) 아직 서버에 올라간 상태는 아님.
	- 메세지는 짧지만 매우 명확하게 (걍 함 ㅇㅇ -> 이딴거 ㅅㅂ)
6. git pull
	- 그새 누가 뭐 작업 했을수도 있으니까 원래 push 전에 pull하는게 정석임.
	- conflict 나면 카톡해...ㄷㄷ
7. git push
	- 서버에 실제로 올림
